Deployment Mode

How many AWS accounts are used for migration?

Single Account

Migration workloads in one AWS account. Deploy the stack once. Done in 2 minutes.

By default all new resources in the account are tagged. Use VPC scoping if migration and non-migration workloads share the same account.
These services have no VPC association. Check to tag them all regardless of VPC scope (recommended for MAP eligibility).
Please enter at least one VPC ID.

Multiple Accounts (AWS Organizations)

Customer has multiple linked accounts under an AWS Organization. One StackSet command deploys to all accounts simultaneously.

The Lambda is deployed to every account in your org via the root OU. By default, all accounts' resources are tagged. You can optionally limit tagging to specific accounts below.
ⓘ This deployment creates a temporary S3 bucket (auto-map-tagger-{account-id}) to stage CloudFormation templates for the StackSet. The bucket is retained after deployment as the StackSet requires it to deploy to new accounts that join your organization in the future.
Please enter at least one account ID.
