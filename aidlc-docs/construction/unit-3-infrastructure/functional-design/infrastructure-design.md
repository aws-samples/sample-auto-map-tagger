# Infrastructure Design — Unit 3: Infrastructure

## Region & Account
- **Deployment**: Customer's AWS account(s)
- **Multi-account**: From management account via StackSet
- **IaC**: CloudFormation (embedded in deploy.sh)

## CloudFormation Resources

### Event Capture
| Resource | Type | Config |
|---|---|---|
| EventBridge Rule | `AWS::Events::Rule` | Matches CloudTrail creation events, filtered by scope |
| EventBridge Target | — | SQS queue |

### Buffering
| Resource | Type | Config |
|---|---|---|
| SQS Queue | `AWS::SQS::Queue` | 14-day retention, 180s visibility timeout |
| Dead Letter Queue | `AWS::SQS::Queue` | maxReceiveCount: 5 |
| Redrive Policy | — | Route to DLQ after 5 failed receives |

### Compute
| Resource | Type | Config |
|---|---|---|
| Lambda Function | `AWS::Lambda::Function` | Python 3, SQS event source |
| Event Source Mapping | `AWS::Lambda::EventSourceMapping` | SQS → Lambda, batch size, partial batch response |

### Permissions
| Resource | Type | Config |
|---|---|---|
| Lambda Execution Role | `AWS::IAM::Role` | Least-privilege: tag:*, sqs:*, ssm:GetParameter, logs |
| StackSet Admin/Exec Roles | `AWS::IAM::Role` | Cross-account deployment (org mode) |

### Config & Alerting
| Resource | Type | Config |
|---|---|---|
| SSM Parameter | `AWS::SSM::Parameter` | Scope config (MPE, dates, accounts, VPCs) |
| CloudWatch Alarm | `AWS::CloudWatch::Alarm` | DLQ ApproximateNumberOfMessagesVisible > 0 |
| SNS Topic | `AWS::SNS::Topic` | Alert notifications |
| SNS Subscription | `AWS::SNS::Subscription` | Email endpoint |

## Deployment Modes

### Single-Account (Stack)
```
deploy.sh → aws cloudformation deploy → Stack: map-auto-tagger-mig<MPE>
```

### Multi-Account (StackSet)
```
deploy.sh → aws cloudformation create-stack-set
         → create-stack-instances (across ScopedAccountIds)
         → StackSet: map-auto-tagger-mig<MPE>
```

## IAM Least-Privilege Policy (generated)

```json
{
  "Effect": "Allow",
  "Action": [
    "tag:TagResources",
    "ec2:CreateTags",
    "rds:AddTagsToResource",
    "... per-service tagging actions ..."
  ],
  "Resource": "*"
}
```

Generated by `.github/scripts/generate_iam.py` to include only the tagging actions needed for the 154 supported services.

## Cost Model

| Component | Monthly (per account) |
|---|---|
| Lambda (100-1000 invocations/day) | $0.10 – $2.00 |
| EventBridge + SQS + SSM | $0.01 – $0.20 |
| **Total** | **< $2/month** |

## Security Considerations
- No inbound network exposure (all event-driven, internal)
- Least-privilege IAM (tagging only, no read of resource data)
- Customer owns all deployed roles
- Generated scripts linted for shell injection (CI)
